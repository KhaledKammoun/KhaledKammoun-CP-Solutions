Problem G — Olivesoft's Irreducible Cores

Time Limit: 1.00 s
Memory Limit: 256 MB
Olivesoft is stress-testing their new AI data-lake architecture. The pipeline processes n distinct data batches concurrently, where the i-th batch contains exactly ai data packets.

During the compression phase, Olivesoft engineers must identify what they call Irreducible Core batches. To determine if a batch size x is an Irreducible Core, the system calculates its Cyclic Resilience Score, denoted as R(x).

The system calculates R(x) by computing the factorial of (x - 1) and finding its remainder when divided by x. Formally:

R(x) = (x - 1)! mod x

Through rigorous testing, Olivesoft defined that a batch size x (where x > 1) is an Irreducible Core if and only if its score is exactly equal to x - 1.

For example, for a batch of size 4: R(4) = 3! mod 4 = 6 mod 4 = 2. Since 2 ≠ 3, a batch of 4 is not an Irreducible Core.

Given the sizes of the n data batches, your task is to write a program that calculates the total number of Irreducible Core batches currently in the pipeline.

Input

The first line contains a single integer n — the number of data batches.

The second line contains n space-separated integers a1, a2, ..., an — the number of data packets in each batch.

Output

Print a single integer — the total number of Irreducible Core batches.

Constraints

  • 1 ≤ n ≤ 106
  • 1 ≤ ai ≤ 106

Example

Input 
10
2 4 5 6 7 8 9 10 11 1
Output 
4

Note

Let's simulate the Cyclic Resilience Score for a few elements from the example:

  • x = 2 → R(2) = 1! mod 2 = 1. (Score equals 2 - 1) → Irreducible
  • x = 5 → R(5) = 4! mod 5 = 24 mod 5 = 4. (Score equals 5 - 1) → Irreducible
  • x = 6 → R(6) = 5! mod 6 = 120 mod 6 = 0. (Score is 0 ≠ 5)
  • x = 11 → R(11) = 10! mod 11 = 3628800 mod 11 = 10. (Score equals 11 - 1) → Irreducible
  • x = 1 → Ignored, as Irreducible Cores must be strictly greater than 1.
