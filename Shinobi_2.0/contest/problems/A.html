Problem A — The Piximind Talent Stream Analysis

Processing Time: 1.00 s
Memory Capacity: 512 MB
At Piximind, recruitment is viewed through the lens of data science. Every ISIMS student who applies is part of a chronological "Talent Stream" that has been recorded since the company's founding. To maintain a culture of innovation, the engineering leads believe in the "Principle of Cognitive Variety."

The principle states that a team’s problem-solving capacity is highest when every member possesses a unique "Logic Signature." In the Piximind database, these signatures are stored as integer Logic IDs. When analyzing the history of n applicants, the HR team wants to identify the most successful period of recruitment—defined as the longest continuous sequence of students where no two individuals shared the same Logic ID.

As an analyst, you are given the full chronological stream. Your goal is to find the length of the Longest Variety Batch. If a student appears in the stream with an ID that has already been seen within the current batch, the "variety" is lost, and the batch must be considered terminated at that point.

Because Piximind has grown rapidly, the stream contains hundreds of thousands of entries. A slow, manual comparison of every possible batch will be inefficient. You must develop a solution that can process the entire talent stream in a single pass to ensure the data is analyzed before the next recruitment cycle begins.

Input Protocol

  • The first line contains an integer n: the total number of ISIMS applicants recorded in the Piximind stream.
  • The second line contains n integers k1, k2, ..., kn: the Logic ID assigned to each student in the exact order they applied.

Output Protocol

Print a single integer: the maximum length found for a continuous sequence of applicants with unique Logic IDs.

Operational Constraints

  • The number of students n can reach up to 2 · 105.
  • Each Logic ID ki is a value between 1 and 109.
  • The system must complete the analysis within 1.00 second.

Case Study

Input Data 
8
1 2 1 3 2 7 4 2
Analysis Result 
5

Intuition & Observation

Consider the stream [1, 2, 1, 3, 2, 7, 4, 2]. We track the current batch by observing how it evolves as new students apply:

  • The stream starts with [1, 2]. Variety is maintained.
  • The next ID is 1. This is a duplicate of our first applicant. The current "Unique Batch" starting from index 0 is broken.
  • A new variety batch can be found starting after the first 1, leading to the sequence [1, 3, 2, 7, 4].
  • All IDs in [1, 3, 2, 7, 4] are distinct, giving us a length of 5.
  • The next ID in the stream is 2, which already exists in our current batch, meaning the maximum unique span for this section ends here.
