Problem D — PROGED Azure Data Stream

Processing Time: 1.00 s
Memory Capacity: 512 MB
At PROGED Solutions, cloud engineers build high-speed data pipelines using Microsoft Azure. The system processes a continuous stream of data packets, where each packet is tagged with a specific Data ID.

Azure has a strict rule for data quality: A continuous sequence of packets is called a Clean Batch only if every single packet inside it has a different Data ID. If any ID repeats within the sequence, the entire batch is rejected.

As the Lead Data Engineer, you are monitoring a stream of n packets. Your task is to calculate the total number of valid "Clean Batches" (subarrays) that can be formed. Remember, every unique pair of starting and ending positions in the stream counts as a distinct valid batch, as long as all IDs inside that range are unique.

Input Protocol

  • The first line contains an integer n: the total number of data packets.
  • The second line contains n integers k1, k2, ..., kn: the Data ID stamped on each packet in the order they arrive.

Output Protocol

Print a single integer: the total number of continuous segments (subarrays) that contain entirely unique Data IDs.

Operational Constraints

  • The number of packets n can reach up to 2 · 105.
  • Each Data ID ki is a value between 1 and 109.

Engineer's Example

Input Data 
4
1 2 1 3
Engineer's Result 
8

Intuition & Observation

Why does the data stream [1, 2, 1, 3] yield exactly 8 Clean Batches?

  • Length 1: [1], [2], [1], [3] (4 valid).
  • Length 2: [1, 2], [2, 1], [1, 3] (3 valid. Note: [1, 2] and [2, 1] are distinct by their positions).
  • Length 3: [2, 1, 3] (1 valid. [1, 2, 1] is rejected because ID 1 repeats).
  • Length 4: [1, 2, 1, 3] (0 valid, rejected because ID 1 repeats).

Total valid batches: 4 + 3 + 1 = 8.

Engineer's Note: Instead of checking every starting position from scratch, maintain a sliding "safe zone" from left to right. If your current zone contains no duplicates, the number of new valid batches ending at the newly added rightmost packet equals the total number of packets currently inside that safe zone!
